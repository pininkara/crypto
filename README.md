# 加密货币工具箱项目 (Crypto Toolbox)

本项目使用 **Golang + Gin + Telebot** (后端) 及 **React + TypeScript + Vite + TailwindCSS** (前端) 构建，提供两大核心功能：
1. **Telegram 币价监控**：支持通过机器人配置自动化提醒，当 Binance 现价穿越你的目标价格时会自动推送 Telegram 提醒消息。
2. **加密网格策略回测计算器**：基于 web 界面，提供交互式表单计算指定网格参数的历史(近期 30 天)回测利润预估。

## 功能介绍

### 1. Telegram 机器人监控终端
需要在 Telegram 中向您的机器人发送如下快捷指令 (控制台已全部支持纯英文输出指引)：
- `/start` 或 `/help` 显示操作帮助菜单。
- `/add <symbol> <price>` (例如: `/add btc 80000`) 创建实时价格监控点。
- `/list` 列出所有生效中的警报（采用全新的排版，实时展示资产现价，并自动通过 📈和📉 表情符号展示相对于目标价的趋势与差值）。
- `/delete <id>` 根据编号精准删除特定任务。
- `/deleall` 一键暴力清空名下所有监控项。

### 2. Crypto Toolbox (全能 Web 界面)
访问本服务的根网址（例如 `http://127.0.0.1:8080/`），即可体验带有独立选项卡切换的 Web 3.0 工作台：
- **高级流光视觉**：支持深浅双色系统，全局背景更采用高达 8 色相融合的 30 秒缓慢呼吸态纯原生 CSS 渐变动画。
- **高频多时段回测**：彻底告别失真的粗略估算！系统借助高并发引擎，针对不同时间窗口采用最高级别的 K 线并发模拟：
  - **每日利润 / 逃逸率**：穿透查询最近 24 小时的 `5m` 级别 K 线（288根），捕捉极其细微的插针套利利润。
  - **每周利润 / 逃逸率**：扫描最近 7 天的 `15m` 级别 K 线（672根）。
  - **每月利润 / 逃逸率**：基于最近 30 天由 `1h` 级别组成的全量 K 线回测。
- **动态风险追踪**：计算系统根据 K 线的 High/Low 极值，精算历史表现中价格直接飞出网格上下限的“逃逸率”。并以绿（健康 <15%）、黄（警告 <35%）、红（瘫痪 >35%）的色控直观反显策略风险。

## 运行与部署 (推荐利用 Docker)

整个系统前后端均被打包至一个单独的轻量 Alpine 容器中。

### a) 准备配置环境
在根目录下新建 `config` 文件夹，并复制示例配置：
```bash
mkdir config
cp config.example.toml config/config.toml
```
编辑 `config/config.toml`：
- 在 `[telegram]` 下输入 `token = "你的机器人Token"`
- 在 `[telegram]` 下输入 `allowed_users = [你的TelegramUID]`（如果填写，则只有这些 ID 才可以使用你的机器人）。
- 在 `[telegram]` 下配置 `monitor_interval = 30` 或更高以定义监控币价防频繁发包的轮询秒数。
- 在 `[binance]` 下如果不想填可以留空（读取公开现货和K线无需鉴权）。

### b) 准备数据目录
确保当前目录下有一个可读写的 `data` 文件夹（用于存放 `crypto.db` SQLite 数据持久化）。
```bash
mkdir data
```

### c) 一键启动
```bash
docker-compose up -d --build
```
此时 Web 服务将抛出在宿主机的 `8080` 端口。你可以访问 `http://localhost:8080` 进行测试。

## 开发指南
**前端子模块：**
1. `cd web`
2. `npm install`
3. `npm run dev` 运行前端开发服务器
4. `npm run build` 打包出 `/dist` 目录

**后端主程序：**
1. 保证处于主目录下并存有正确 `config.toml` 文件或通过环境变量加载。
2. `go run ./cmd/server` 启动本地 API 以及机器人长轮询协程。
